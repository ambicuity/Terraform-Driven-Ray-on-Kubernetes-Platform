# Contributing Guidelines

Thank you for your interest in contributing to the **Terraform-Driven Ray on Kubernetes Platform**! We welcome bug reports, feature requests, and pull requests.

## How to Contribute

1.  **Find an Issue:** Look for open issues. If you are new to the project, check out the issues labeled [`good first issue`](https://github.com/ambicuity/Terraform-Driven-Ray-on-Kubernetes-Platform/issues?q=is%3Aissue+is%3Aopen+label%3A"good+first+issue") or [`help wanted`](https://github.com/ambicuity/Terraform-Driven-Ray-on-Kubernetes-Platform/issues?q=is%3Aissue+is%3Aopen+label%3A"help+wanted").
2.  **Autonomous AI Engineering Organization:** Note that this repository employs highly autonomous AI Agents (Gamma, Delta, Beta, Alpha) that automatically triage issues, generate code, review PRs, and manage releases. Ensure your issues are well-structured (with "Steps to Reproduce", "Expected Behavior") to help Agent Gamma successfully triage them.
3.  **Fork and Branch:** Fork the repository and create a descriptive branch for your feature or bug fix.
4.  **Local Development:**
    *   Ensure any new Python automation scripts conform to PEP8.
    *   Validate YAML syntax for GitHub Actions and Terraform configurations.
    *   Run local tests if applicable.
5.  **Submit a Pull Request:** Submit your PR against the `main` branch. Provide a clear description of your changes.
6.  **Review your PR with Gemini Code Assist:** This repository uses Gemini Code Assist on GitHub for code reviews. In your PR comment thread, you can use the commands `/gemini summary` to get an instant summary, and `/gemini review` to get a deep code review of your changes.

## Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct and remain respectful to all human and AI collaborators.

## Reporting Bugs

Please use the GitHub Issue tracker to report bugs. Include a clear and descriptive title, steps to reproduce, and the environment (OS, Terraform Version, Ray Version) where the bug occurred.
